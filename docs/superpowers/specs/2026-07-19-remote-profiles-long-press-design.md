# RC003 应用配置与长按动作设计

日期：2026-07-19

## 目标

让 RC003 在 Codex、Claude Code 和其他 macOS 应用中使用合适的按键映射，并允许每个实体按键设置单击和长按动作。现有正式 App 保持不变，本次交付覆盖旁装的“快捷键测试”App。

## 配置选择

App 在收到实体按下事件时读取前台应用的 bundle identifier，并选择一次配置：

1. `com.openai.codex` 使用 Codex 配置。
2. 用户勾选的终端宿主使用 Claude Code 配置。
3. 其他应用使用通用配置。

一次按住期间保持按下时选中的配置。设置页提供以下 Claude Code 宿主候选：Terminal、Ghostty、Warp、iTerm、Visual Studio Code 和 Cursor，默认勾选 Ghostty 与 Warp。

## 按键模型

每个配置保存 12 个 RC003 按键，每个按键包含：

- 单击动作：必填，沿用现有预置动作或自定义快捷键。
- 长按动作：可选，沿用同一套动作类型。

麦克风键继续控制现有语音录制，不进入这套按键配置。

### 事件规则

- 没有长按动作：按下时立即执行单击动作，并保留当前方向、返回和音量键连发。
- 有长按动作：按下后等待 1 秒。1 秒内松开只执行单击动作；达到 1 秒只执行长按动作，松开时不再执行单击。
- 方向键设置长按动作后停用该按键的连发。
- 设备断开、权限失效、关闭映射或停止监听时取消全部待执行动作。
- 同一个实体按下周期最多执行单击或长按中的一个动作。

## 默认映射

方向键在三个配置中均为方向操作，电源键均为 Escape。未列出的长按动作默认禁用。

| 按键 | 通用单击 | 通用长按 | Codex 单击 | Codex 长按 | Claude Code 单击 | Claude Code 长按 |
|---|---|---|---|---|---|---|
| OK | Enter | 无 | Enter | 无 | Enter | `Ctrl-J` 换行 |
| 返回 | Delete | `Cmd-Delete` 清行 | Delete | `Cmd-Delete` 清行 | Delete | `Ctrl-U` 清行 |
| 音量 + | 系统音量 + | 无 | `Cmd-Shift-[` 上一个会话 | `Cmd-[` 后退 | `Ctrl-R` 历史搜索 | `Ctrl-B` 后台运行 |
| 音量 - | 系统音量 - | 无 | `Cmd-Shift-]` 下一个会话 | `Cmd-]` 前进 | `Ctrl-T` 任务清单 | `Ctrl-S` 暂存输入 |
| 主页 | 显示桌面 | 无 | `Cmd-N` 新建会话 | `Cmd-Shift-P` 命令菜单 | `Ctrl-C` 中断 | `Option-O` 快速模式 |
| 菜单 | 右键菜单 | 无 | `Ctrl-Shift-G` 打开审查 | `Cmd-Option-B` 切换审查面板 | `Shift-Tab` 权限模式 | `Option-T` 思考模式 |
| TV | 应用切换 | 无 | `Cmd-G` 搜索会话 | `Ctrl-Tab` 最近会话 | `Ctrl-O` 详细记录 | `Option-P` 模型选择 |

审批、拒绝、删除任务、提交和推送不进入默认映射。用户仍可把已有快捷键手动配置为长按动作。

## 设置与迁移

设置页使用“通用 / Codex / Claude Code”分段选择器。每一行同时显示单击和长按配置，并分别提供快捷键录制按钮。页面显示当前前台应用会使用的配置，并提供 Claude Code 宿主复选框。

首次读取新版配置时：

1. 将当前 `buttonBindings` 完整迁移到通用配置。
2. Codex 和 Claude Code 使用本设计中的默认值。
3. 已有自定义映射开关保持原值。
4. 单个损坏的按键或配置只恢复对应默认值，不丢弃其他有效配置。

## 模块与接口

- 配置选择模块：输入前台应用标识和已勾选宿主，返回通用、Codex 或 Claude Code。
- 按键手势模块：输入按下、达到阈值、松开和取消事件，输出零个或一个按键动作。
- 设置模块：读取、迁移和保存三个配置及宿主选择。
- HID 监听模块：把实体事件交给按键手势模块，并把输出交给现有 `KeyboardInjector`。

配置选择和按键手势使用纯逻辑接口，测试不需要真实 HID 设备、计时器或前台应用。

## 验证

自动测试覆盖以下接口：

- 前台应用标识选择正确配置。
- 有长按动作时，短按、长按和取消分别产生正确结果。
- 无长按动作时立即执行，并保持现有连发规则。
- 旧配置迁移、三个配置独立保存、单项损坏恢复。
- 自定义组合键包含明确的修饰键按下与松开事件。

交付前运行 `./scripts/test.sh`、`xcrun swift test`、发布构建和 App 验证脚本。随后覆盖 `/Applications/小米遥控器桥接-快捷键测试.app`，保留 `/Applications/小米遥控器桥接.app`。

## 本次不包含

- 多段按键宏，例如依次发送两个不同快捷键。
- 根据终端标签页内容判断 Claude Code 是否正在运行。
- 审批和发布类动作的默认映射。
- BlackHole 安装包与自动声音选择；该工作在按键功能验证后单独实施。
