小米遥控器桥接 0.2.0 测试版
============================

重要说明
--------

这是供小范围测试的 Universal 应用，支持 Apple Silicon 和 Intel，最低部署目标为 macOS 11 Big Sur。
当前安装包使用 ad-hoc 签名，没有 Apple Developer ID，也没有经过 Apple 公证。它不是正式签名发行版。
macOS 11 部署目标已经双架构编译校验，但尚未在真实 macOS 11 机器上运行验收。

首次安装
--------

1. 把“小米遥控器桥接.app”拖到旁边的 Applications。
2. 在“应用程序”中按住 Control 单击应用，选择“打开”。
3. 如果系统仍然阻止打开，进入“系统设置/系统偏好设置 -> 隐私与安全性”，确认应用来自你信任的分享者后选择“仍要打开”。
4. 按提示允许蓝牙。启用自定义按键映射时，还需要允许“输入监控”和“辅助功能”，然后完全退出并重新打开应用。

遥控器配对
----------

1. 在 Mac 蓝牙设置中打开蓝牙。
2. 同时长按遥控器“主页”和“菜单”键，使遥控器进入配对广播状态。
3. 在 Mac 上接受“小米蓝牙语音遥控器 / MI RC / Xiaomi Bluetooth Remote 2 Pro”的连接提示。

语音与 Fn
---------

1. 安装 BlackHole 2ch 或等价的双向 CoreAudio 回环设备。
2. 在本应用“虚拟麦克风 -> 语音输出”中选择该回环设备。
3. 在豆包输入法或其他目标语音应用的“麦克风选择”中选择同一个回环设备。只选“MacBook Air 麦克风”不会使用遥控器传来的语音。
4. 先单击目标输入框，确认竖线光标已经出现。
5. 按住遥控器麦克风键说话：应用会把该遥控器真实上报的 F5 硬件按下/松开映射为 Mac Fn，同时把 RC003 语音送入回环设备；松开时会直接释放 Fn 并结束语音流。该映射只匹配小米 RC003，不修改 MacBook 键盘或其他设备，应用退出时恢复原映射。

豆包输入法若只出现绿色提示、没有黑色波纹，通常表示它没有取得当前可编辑输入框，只能退回复制/粘贴。请松开按键、重新单击输入框，确认光标出现后再试。如果在 TextEdit 等系统应用中长按电脑实体 Fn 也只能复制，则是该输入法当前的直接写入问题，不代表遥控器或桥接断线；可切换到实体 Fn 测试能直接上屏的输入法再使用遥控器。

隐私与边界
----------

- 应用不上传语音，不保存语音文件；PCM 只在内存与用户选定的音频设备之间流动。
- 应用不会修改系统默认输入或输出设备。
- 应用日志不记录语音内容、蓝牙地址或外设 UUID。
- 实物图由项目用户提供；程序的 GPL-3.0-only 许可不额外授予图片或 Xiaomi 商标的转载权。

卸载
----

退出应用后删除“应用程序/小米遥控器桥接.app”。如需移除权限记录，可在系统“输入监控”和“辅助功能”中删除对应条目。
